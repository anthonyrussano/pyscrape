MongoDB Docker Compose

June 10, 2023
can you show me a docker compose file for running the latest version of mongodb

Certainly! Here's an example of a Docker Compose file that can be used to run the latest version of MongoDB: 

yaml
version: '3.8'
services:
  mongodb:
    image: mongo:latest
    restart: always
    ports:
      - 27017:27017
    volumes:
      - mongodb_data:/data/db
    environment:
      - MONGO_INITDB_ROOT_USERNAME=admin
      - MONGO_INITDB_ROOT_PASSWORD=pass123

volumes:
  mongodb_data:

In this Docker Compose file, we define a service called mongodb based on the mongo:latest image, which pulls the latest version of MongoDB from the official Docker Hub repository. Here are some key points about the configuration:

  • The restart: always option ensures that the MongoDB container restarts automatically if it stops for any reason.
  • The ports section maps the container's port 27017 to the host's port 27017, allowing you to access MongoDB from your local machine.
  • The volumes section creates a named volume called mongodb_data and mounts it to the container's /data/db directory. This allows MongoDB to store its data persistently, even if the container is stopped or deleted.
  • The environment section sets the root username and password for the MongoDB instance. You can modify these values as per your requirements.

To run this Docker Compose file, save it as docker-compose.yml and execute the following command in the directory where the file is located: 

shell
docker-compose up -d

This command will start the MongoDB container in detached mode, allowing it to run in the background. You should now have a MongoDB instance running on your local machine, accessible on localhost:27017 .

Remember to adjust the file permissions and secure the MongoDB instance appropriately before using it in a production environment.

this is determined to be due to my cpu does not support avx instructions

If you're encountering an issue where your CPU does not support AVX instructions while trying to run the latest version of MongoDB in a Docker container, it's possible that the MongoDB Docker image you're using requires AVX support, which might not be available on your CPU.

In such a case, you have a couple of options:

  1. Use an older version of the MongoDB Docker image: Instead of using the latest version, you can specify a specific version of the MongoDB image that is compatible with your CPU. You can browse the Docker Hub repository for the available versions and choose one that suits your needs.

  2. Build a custom MongoDB image: You can create a custom Docker image for MongoDB, modifying the Dockerfile to build MongoDB from source or use an alternative base image that does not have AVX instruction requirements. This approach requires some knowledge of Dockerfile and MongoDB installation.

Here's an example of a Dockerfile that uses an alternative base image and installs MongoDB: 

dockerfile
FROM debian:buster

# Install MongoDB dependencies
RUN apt-get update && apt-get install -y wget gnupg2
RUN wget -qO - https://www.mongodb.org/static/pgp/server-5.0.asc | apt-key add -
RUN echo "deb http://repo.mongodb.org/apt/debian buster/mongodb-org/5.0 main" | tee /etc/apt/sources.list.d/mongodb-org-5.0.list
RUN apt-get update && apt-get install -y mongodb-org

# Create MongoDB data directory
RUN mkdir -p /data/db

# Expose MongoDB port
EXPOSE 27017

# Start MongoDB
CMD ["mongod"]

With this Dockerfile, you can build a custom MongoDB image using the following command: 

shell
docker build -t my-mongodb-image .

After building the image, you can modify your Docker Compose file to use your custom image instead of the official MongoDB image: 

yaml
version: '3.8'
services:
  mongodb:
    image: my-mongodb-image
    restart: always
    ports:
      - 27017:27017
    volumes:
      - mongodb_data:/data/db

volumes:
  mongodb_data:

Save the updated Docker Compose file and run it again using docker-compose up -d .

By using an alternative base image and building a custom MongoDB image, you can work around the AVX instruction requirement and run MongoDB on your CPU.

how can i switch to mongodb v4

To switch to MongoDB version 4.x in your Docker Compose setup, you need to specify the desired version of the MongoDB image in your Docker Compose file. Here's an example of a Docker Compose file that runs MongoDB version 4.x: 

yaml
version: '3.8'
services:
  mongodb:
    image: mongo:4
    restart: always
    ports:
      - 27017:27017
    volumes:
      - mongodb_data:/data/db
    environment:
      - MONGO_INITDB_ROOT_USERNAME=admin
      - MONGO_INITDB_ROOT_PASSWORD=pass123

volumes:
  mongodb_data:

In this updated Docker Compose file, the image field for the MongoDB service is set to mongo:4 , which refers to the MongoDB version 4.x image.

Save the updated Docker Compose file, and then you can start the MongoDB container running MongoDB version 4.x using the following command: 

shell
docker-compose up -d

This will start the MongoDB container in the background, and you can access it on localhost:27017 .

Please note that the specific tag 4 refers to the latest available version within the 4.x series at the time of pulling the image. If you require a specific minor version, you can specify it explicitly using the full image tag, such as mongo:4.4 for MongoDB version 4.4.

Remember to adjust the configuration and security settings as needed for your specific use case.
